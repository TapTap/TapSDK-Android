URL_SAFE

Encode using Base64-like encoding that is URL- and Filename-safe as described in Section 4 ofRFC3548:http://www.faqs.org/rfcs/rfc3548.html.It is important to note that data encoded this way is not officially valid Base64, orat the very least should not be called Base64 without also specifying that is was encoded usingthe URL- and Filename-safe dialect.
